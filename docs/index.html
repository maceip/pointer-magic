Pointer Magic

A local mac app that rides beside your real cursor. No cloud account. No cursor replacement. One menu bar app that can watch your coding agents, or show context for whatever is under the pointer.

Install on this Mac Two ways to use it GitHub

One shelf. Every agent. Right under your hand.

If you clone this today

There is no website signup and no setup wizard. You build a small menu bar app on your Mac, grant permissions, and it starts watching locally. That is intentional, and it is also what we need to explain clearly for people who are not in a terminal agent all day.

What happens

After the build, pointer magic opens, a menu bar icon appears, and the app sits quietly. Without permissions it cannot follow the pointer well. Without coding agents, the agent shelf stays empty, but the under-pointer context path can still work.

What we want to happen

In under five minutes: build the app, grant permissions, then see either your agents readiness or a shelf of local context when you pause the pointer. No config file. No api keys for the core app.

Install

You need a Mac on macOS 14 or newer, and apples command line tools or xcode so the swift command works. That is the only setup.

Clone and build the app

From terminal:

git clone https://github.com/maceip/pointer-magic.git
cd pointer-magic/apps/pointer-magic-macos
./scripts/build-app.sh --open

Find the menu bar icon

Look for the pointer magic icon near the clock. Left-click parks the shelf. Right-click opens the menu for permissions, on or off, and quit.

Grant permissions one at a time

Open the menu and choose request next permission. Repeat after each toggle in system settings. Usual order: input monitoring, then accessibility, then screen recording. Automation is only needed if you want click shelf to jump to that agents terminal.

Turn pointer magic on

In the menu, leave pointer magic checked. Move the mouse and pause. You should not see a new system cursor, only an optional companion shelf when there is something to show.

If you rebuild often and want permissions to keep the same app identity, sign with your local development certificate using the POINTER_MAGIC_CODESIGN_IDENTITY environment variable when you run build-app.sh.

Two clear cases

Pick the story that matches how you work. You do not need both.

Case 1 · Agents

Which agent is ready

You run several local coding agents such as codex, claude code, or cursor. Pointer magic finds their sessions on disk and shows a compact shelf with provider, folder, and state, so you can see who finished, who is waiting, and whose state just changed.

  • Keep agents running in ghostty, terminal, or cursor as usual.
  • When a sessions state changes, the shelf updates while you work.
  • Click to park the shelf, then click it to focus that agents terminal.
  • No api keys inside pointer magic. It only reads local session files.

Case 2 · Everyone else

What is under my pointer

You are not attached to a coding agent. You still get a local shelf when the pointer settles: accessibility text, a small on-device crop and text reading pass, and simple actions from the built-in sample provider. Same app, different job.

  • Grant accessibility, and screen recording if you want text reading.
  • Pause over mail, notes, a browser, or any ordinary app.
  • When context is available, a shelf appears beside the cursor.
  • Park to click actions. Dismiss hides that exact update.

Configuration

There is no day-one config file. Discovery uses the normal local folders for codex, claude, and cursor. Providers are swift modules in the app. See the native docs if you want to write your own shelf.

Built in, zero setup

The agent shelf and sample context provider start with the app. If you have no agents, case 1 stays quiet and case 2 can still work.

For people extending it

Read the shelf providers doc and the native package readme in the github repo under apps/pointer-magic-macos.

Privacy in one paragraph

Pointer magic does not ship a network client for its core loop. Crops and on-device text reading stay on your Mac. Agent discovery reads local session files and does not upload them. Terminal focus uses apple events only after you click a parked shelf.
